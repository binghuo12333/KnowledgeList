# 一、大模型协议分类及核心定义

大模型协议本质是规范模型使用权、修改权、分发权及商用边界的法律文本，核心分为三大类，覆盖不同研发与落地场景：

- **开源协议**：允许自由获取、修改、分发，核心强调开源义务，部分限制商用（如AGPL），代表：Apache 2.0、MIT、GPLv3、AGPLv3。

- **非商业许可协议**：禁止或限制商用，仅允许学术/个人研究使用，代表：Llama 2 Community License（非商用版）、GLM-4 非商用许可。

- **商用许可协议**：明确授权商用场景，部分需付费/申请授权，代表：Llama 2 Commercial License、Claude API 许可、通义千问商用许可。

关键提醒：大模型协议不仅约束模型本身，还可能覆盖衍生作品（微调模型、基于模型开发的应用），需结合具体条款判断合规性。

# 三、协议选型及合规使用说明

## 3.1 场景化选型建议

### 个人/学术研究

- 优先选：Apache 2.0、MIT（无约束，可自由微调）

- 次选：Llama 2 Community License、GLM-4 非商用版（免费但禁止商用）

- 避坑：AGPLv3（网络服务需开源，研究场景无影响但后续商用迁移麻烦）

### 企业商用（闭源产品）

- 优先选：Apache 2.0、MIT（允许闭源分发衍生作品，无商用限制）

- 次选：付费商用许可（Llama 2 商用版，适合追求模型性能场景）

- 避坑：GPLv3/AGPLv3（强制开源衍生作品，不符合闭源产品需求）

### 企业商用（开源产品）

- 优先选：GPLv3（适合客户端开源产品）、Apache 2.0（灵活度高）

- 次选：AGPLv3（仅适合网络服务类开源产品）

### 网络服务部署（如API服务）

- 优先选：Apache 2.0、MIT（无开源强制要求，可闭源部署）

- 避坑：AGPLv3（强制开源服务端源码，泄露技术细节）

## 3.2 核心合规要点

1. **版权声明保留**：所有协议均要求保留原模型的版权声明和许可文本，即使微调后也需在产品文档/源码中注明。

2. **衍生作品标注**：Apache 2.0要求对修改部分标注修改记录，避免与原模型混淆。

3. **商用边界界定**：“商用”包括盈利性应用、企业内部业务使用、付费服务等，非商用许可（如GLM-4非商用版）禁止此类场景。

4. **数据合规补充**：协议仅约束模型使用权，使用模型时需额外遵守数据隐私法规（如GDPR、个人信息保护法），禁止输入敏感数据。

# 四、实操示例：不同协议模型的使用流程

## 4.1 Apache 2.0 模型（以Qwen-2-7B-Chat为例）

### 适用场景：企业闭源商用产品、网络API服务

#### 步骤1：模型下载（合规获取）

```bash

# 从魔搭下载（国内合规渠道）
pip install modelscope
modelscope download --model=qwen/Qwen-2-7B-Chat --local-dir=./Qwen-2-7B-Chat
# 保留模型根目录下的LICENSE文件（Apache 2.0文本）
```

#### 步骤2：微调与修改（合规改造）

- 可自由微调模型，适配企业业务场景（如行业知识库注入）。

- 在微调后的模型文档中注明：“基于Qwen-2-7B-Chat（Apache 2.0协议）微调，原版权归Alibaba Cloud所有”。

#### 步骤3：商用部署（合规分发/服务）

```bash

# 部署闭源API服务（无需开源微调代码）
python -m fastchat.serve.openai_api_server --model-path ./Qwen-2-7B-Chat --port 8000
# 合规要求：在API服务文档中添加版权声明和许可链接
```

## 4.2 AGPLv3 模型（以LLaVA为例）

### 适用场景：开源网络服务、学术研究

#### 步骤1：模型下载

```bash

huggingface-cli download liuhaotian/LLaVA-7B-v1.5 --local-dir=./LLaVA-7B
# 保留LICENSE文件（AGPLv3文本）
```

#### 步骤2：修改与部署

- 微调模型后，需将微调代码开源（如上传至GitHub，标注AGPLv3协议）。

- 部署网络服务时，需在服务首页添加“源码下载链接”，指向微调代码仓库。

#### 步骤3：合规分发

分发微调后的模型时，需同时提供：1. 模型文件；2. 全部微调源码；3. AGPLv3许可文本。

## 4.3 非商用许可模型（以GLM-4-7B为例）

### 适用场景：个人研究、学术课题（禁止商用）

#### 步骤1：获取许可与模型

- 在魔搭社区签署《GLM-4非商用许可协议》后，方可下载模型。

- 下载命令：`modelscope download --model=ZhipuAI/glm-4-7b --local-dir=./GLM-4-7b`

#### 步骤2：合规使用

- 仅可用于学术论文、个人实验，禁止用于企业内部业务、盈利性应用。

- 禁止修改模型标识，禁止将模型输出用于商用内容生成。

违规提示：若将GLM-4非商用版用于商用场景，Zhipu AI有权追究法律责任，包括停止使用、赔偿损失。

## 4.4 API商用许可（以GPT-3.5为例）

### 适用场景：企业商用产品（无模型源码，仅API调用）

#### 步骤1：获取API密钥（合规授权）

- 注册OpenAI账号，开通付费服务，获取API密钥（自动签署API许可协议）。

#### 步骤2：合规调用

```python

import openai

openai.api_key = "你的API密钥"
response = openai.ChatCompletion.create(
  model="gpt-3.5-turbo",
  messages=[{"role": "user", "content": "生成企业产品文案"}]
)
# 合规要求：1. 不缓存API输出替代调用；2. 不用于生成违法内容；3. 按调用量付费
```

#### 步骤3：商用落地

可将API调用集成到商用产品中，但需在产品隐私政策中注明“使用OpenAI API提供服务”，并遵守数据处理条款（禁止上传敏感数据）。

# 五、常见问题与避坑指南

## 5.1 常见违规场景

1. 将非商用许可模型（GLM-4非商用版、Llama 2 Community版）用于企业盈利性业务。

2. 基于AGPLv3模型部署闭源API服务，未提供源码下载链接。

3. 微调Apache 2.0模型后，删除原版权声明和许可文本。

4. 缓存API输出内容，替代后续API调用（违反OpenAI/Claude API许可）。

## 5.2 避坑技巧

1. 下载模型后，第一时间保留根目录的LICENSE文件，避免后续丢失。

2. 企业商用前，由法务团队审核协议条款，确认无商用限制。

3. 微调模型时，单独记录修改日志，便于满足Apache 2.0等协议的标注要求。

4. 优先选择Apache 2.0/MIT协议模型，兼顾商用灵活性和合规成本。

# AI产品核心设计要点（从0到1关键）
AI产品的设计逻辑与传统产品截然不同，核心优先级为**「效果设计」＞「功能设计」＞「交互设计」**，从0到1设计时需聚焦**提示词设计、输入输出设计、工具调用设计、人机协同设计、错误处理设计、数据闭环设计**6大核心要点，确保产品效果达标、体验流畅、落地可用。

## 3.1 提示词设计：AI产品的「核心产品设计」
提示词是连接用户与大模型的桥梁，也是AI产品经理的**核心能力**，设计的好坏直接决定模型效果，核心原则：**清晰、明确、标准化、场景化**，核心包含3部分：

### （1）系统提示词（System Prompt）
**核心**：定义模型的**角色、核心任务、行为规则、输出格式**，是提示词的基础，决定模型的整体行为逻辑。
**设计原则**：角色贴合场景、任务清晰无歧义、规则明确可执行、输出格式标准化。

**设计原则**：绑定场景特征、补充场景数据、明确场景内的任务规则。

### 提示词优化技巧
1. 避免模糊表述，用「明确指令」替代「模糊要求」（如用「总结成3点，每点不超过20字」替代「简单总结一下」）；
2. 逐步迭代，从基础提示词开始，根据模型输出效果持续优化规则和细节；
3. 小样本示例，在提示词中加入1-2个示例，让模型更贴合预期输出（少样本学习）；
4. 控制长度，避免提示词过长导致模型忽略关键信息。

## 3.2 输入输出设计：极简、标准化
AI产品的输入输出设计直接影响用户体验和模型效果，核心原则：**输入极简、输出标准化、贴合用户使用习惯**。

### 输入设计
**核心**：减少用户的输入成本，让用户快速、准确地传递需求，同时为模型提供足够的信息，提升输出效果。
1. 标准化输入方式：提供**选择框、模板输入、标签选择**等标准化方式，替代无目的的自由输入（如让用户选择「商品咨询/物流查询/售后处理」后再输入问题）；
2. 输入引导：在输入框添加**提示文字、输入示例**，引导用户规范输入（如「请输入订单号查询物流，示例：123456789」）；
3. 多模态输入：根据场景需求，支持**文本、语音、图片、文件**等多模态输入（如售后处理支持用户上传图片）；
4. 输入校验：对用户输入的内容进行简单校验，过滤无效输入，提示用户补充关键信息（如订单号格式错误、缺少核心参数）。

### 输出设计
**核心**：让用户能快速获取核心信息，输出结果贴合用户的使用场景，同时方便后续的人工处理/数据解析。
1. 标准化输出格式：根据场景设计**列表、表格、段落、卡片**等标准化格式，避免杂乱无章的输出（如物流查询用列表展示物流节点，商品参数用表格展示）；
2. 核心信息突出：将关键信息放在显眼位置（如加粗、标色、置顶），让用户一眼看到；
3. 输出长度控制：根据场景需求控制输出长度，避免冗长（如即时通讯场景简洁回答，专业报告场景详细输出）；
4. 多模态输出：根据输入方式和场景需求，支持**文本、语音、图片、文件**等多模态输出（如文案生成支持导出Word/Excel）；
5. 操作便捷性：在输出结果旁添加**复制、收藏、分享、反馈**等按钮，提升用户操作体验。

## 3.3 工具调用设计：精准、可控
若产品需要大模型调用外部工具（如搜索、数据库查询、API调用、文件解析），设计核心：**明确调用条件、可控调用逻辑、完善容错机制**，避免模型误调用、重复调用、调用失败导致的体验问题。

### 核心设计要点
1. 明确调用条件：定义模型**何时需要调用工具、调用哪种工具**，通过规则或意图识别实现精准触发（如用户问实时信息→调用搜索工具，用户问企业内部数据→调用数据库工具）；
2. 可控调用逻辑：设计工具调用的**优先级、参数规范、返回结果处理方式**，避免模型乱调用工具（如同时需要搜索和数据库查询时，先调用数据库，再调用搜索补充）；
3. 完善容错机制：设计工具调用失败的处理方式，包括**重试机制、切换工具、用户提示**（如搜索工具调用失败→重试1次，仍失败→告知用户「暂时无法获取相关信息，建议稍后再试」）；
4. 调用结果校验：对工具调用的结果进行简单校验，过滤无效结果，确保模型基于有效信息输出答案；
5. 调用过程透明：根据场景需求，向用户展示工具调用过程（如「正在为你查询物流信息...」），提升用户感知。

## 3.4 人机协同设计：边界清晰、介入便捷
核心原则：**AI做擅长的，人类做AI不擅长的**，设计要点围绕「清晰的人机边界、便捷的人工介入、完整的人工修正闭环」展开，确保AI出错时能及时兜底，同时让人工修正的结果反哺模型迭代。

### 核心设计要点
1. 清晰的人机边界：明确划分AI和人类的负责范围，通过**规则、场景、效果指标**触发人机切换（如AI回答准确率＜80%→自动转接人工，用户明确要求人工→直接转接人工）；
2. 便捷的人工介入入口：在AI输出结果的显眼位置，增加**人工客服/人工修正/结果审核**入口，支持用户一键触发，无额外操作成本；
3. 平滑的人机切换流程：设计无缝的人机切换流程，确保用户无需重复描述问题（如人工介入时，自动将用户问题、AI回答、交互记录同步给人工客服）；
4. 人工修正的闭环：设计人工修正流程，将人工修正的结果同步到**数据标注系统**，作为模型迭代的训练数据，让模型学习正确答案，减少后续同类错误；
5. 人工干预的权限控制：针对企业级产品，设计不同的人工干预权限（如普通员工可修正答案，管理员可审核修正结果、配置人机切换规则）。

## 3.5 错误处理设计：友好、可反馈
AI模型必然会出现错误（幻觉、答错、工具调用失败、无答案），错误处理设计的核心：**让用户感知不到糟糕的体验，同时收集错误数据用于模型迭代**，避免因模型错误导致用户流失。

### 核心设计要点
1. 错误反馈友好：避免模型输出「无意义的答案」「编造的答案」，若无法解答、回答错误或工具调用失败，直接**坦诚告知用户**，并提供解决方案（如「该问题我暂时无法解答，将为您转接人工客服」）；
2. 一键错误反馈：在AI输出结果旁，增加**「答案错误」「内容无关」「信息不全」** 等精细化反馈按钮，让用户能快速反馈错误，反馈操作越简单越好；
3. 错误原因记录：收集错误案例时，同步记录**错误类型、业务场景、用户问题、模型答案、用户反馈**，为后续模型调优、提示词优化提供依据；
4. 错误补偿机制：针对模型错误给用户带来的不便，提供简单的补偿机制（如企业级产品提供人工优先服务，C端产品提供积分/优惠券）；
5. 实时错误监控：建立模型错误监控体系，实时监控错误率、错误类型，及时发现并解决高频错误问题。

## 3.6 数据闭环设计：自动化、可落地
数据闭环是AI产品持续提升效果的核心，设计要点：**自动化收集、标准化标注、快速迭代**，确保产品上线后能通过数据持续优化，让效果越来越优。

### 核心设计要点
1. 自动化数据收集：通过**产品埋点、自动化日志**，自动收集用户的输入、模型的输出、用户的反馈、模型的错误案例，无需人工介入，确保数据的全面性和及时性；
2. 标准化数据存储：将收集到的数据按**业务场景、数据类型、错误类型**进行分类存储，建立标准化的数据集，方便后续的数据分析和标注；
3. 标准化数据标注：设计**标准化的标注模板和标注流程**，对错误案例、核心业务场景数据进行精细化标注，形成高质量的微调数据集，标注内容包括问题、正确答案、标注说明；
4. 小步快跑的迭代机制：建立**每周/每月**的快速迭代机制，根据标注数据对模型进行提示词优化、小样本微调，持续提升模型在核心场景的效果；
5. 迭代效果验证：每次迭代后，通过**场景化测试用例**验证迭代效果，确保核心指标有提升，避免迭代导致新的问题；
6. 数据隐私保护：在数据收集、存储、标注、使用的全流程，做好**数据加密、脱敏、权限控制**，确保用户数据隐私安全，符合相关法律法规。
